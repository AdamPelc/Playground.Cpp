get_vwap  source: scalar · 1 element / iter

The C++ source loads a single double from each array, multiplies, and accumulates — one element per iteration. The animation below tracks this source-level behavior.
Compiler note With -O3 -mavx2, GCC actually auto-vectorizes this loop. The emitted assembly uses vmovupd ymm0,[rsi+rax] and vmulpd ymm0,ymm0,[rcx+rax] — the same 256-bit load and 4-wide multiply as the AVX2 intrinsic version. The difference is the reduction: the accumulators stay scalar (xmm1, xmm5), so each of the 4 lanes is folded in element-by-element with vunpckhpd/vextractf128+vaddsd inside the hot loop, instead of using vector vaddpd accumulators that get reduced once at the end.
